# Squash Laboratory (sq-lab)

> ⚠️ **プレリリース (v0.5.0)**: これは知り合い向けのプレリリースです。
> 想定外の不具合が残っている可能性があります。ASISでの提供とし、
> 継続的なサポートは約束できません。

スカッシュのプレイヤー分析データ(`result.json`)を閲覧するための、公開の
静的ビューアです。コーチが選手にデータを見せながら指導するための道具として
作っています。初回アクセス時はサンプルデータの分析結果を表示します。

**データはこの端末から一切送信されません。** 読み込んだ JSON はブラウザ内で
処理されるだけで、サーバーへのアップロードや外部への通信は行いません
(サイト共通のバッジ機能が他サイト一覧を同一オリジンから読み込みますが、
これはあなたが読み込んだ試合データとは無関係な静的ファイルです)。

## 使い方

1. `index.html` をブラウザで開く(GitHub Pages 公開後は URL に直接アクセス)。
   初回はサンプルデータの分析結果がそのまま表示される
2. 自分の `result.json` を見るには、画面上部の「ファイルを読み込む」ボタンを
   押して読込UIを開き、ドロップまたはファイル選択で読み込む

読み込む `result.json` は動画解析ツール(the Analyzer、別リポジトリで開発)の
出力です。本サイト自体は動画解析を行いません。

対応スキーマは `schema_version: "0.2"` のみです。各指標の定義は
[guide.html](guide.html) を、開発者向けの詳細仕様は
[docs/spec-viewer.md](docs/spec-viewer.md) を参照してください。

## 既知の制限

- **交差時のID入れ替わり**: 動画解析ツール側の自動割当は、選手同士が接近・
  交差する瞬間に P1/P2 を取り違えることがあります。誤って割り当てるより
  未割当のまま残す方針のため、一部の断片は「未割当」として除外されます
  (試合情報の該当項目からⓘで詳細を確認できます)。
- 各指標(Distance / Work Rate / T-Dominance / Coverage / Avg Speed)の
  詳細な定義は [guide.html](guide.html) を参照してください。

## アセット

- `assets/subu-pic.png`: ヘッダーロゴの元画像(Tomo管理・透過円形加工済み)
- `assets/subu-pic-header.png`: 上記を128×128にリサイズしたヘッダー表示専用
  コピー(元画像が約4MBあり毎ページ読み込むには重すぎるため)。
  **`subu-pic.png` を差し替えた場合は、このリサイズ済みコピーも
  手動で再生成すること**(Pillow等で128×128程度に縮小)。
  画像が存在しない間もヘッダーが崩れないよう `onerror` で非表示にしている

## キャッシュ対策

`index.html` / `guide.html` の script/link 参照には `?v=<VERSION>` を
付与しており、`js/config.js` の `APP_VERSION` と手動で同期させる運用とする
(ビルドステップが無いため自動連動はしていない)。**バージョンを上げる際は
必ず両方の `?v=` を書き換えること**(コミット前チェック項目)。

## 開発について

- vanilla JS + Canvas、ビルドステップなし(npm 依存なし)
- `demo/demo-match.json` は完全な合成データです(架空のプレイヤー名、
  人工的な位置・指標値)。実際の試合データではありません
- サイト共通のバッジ(左下)は `subutomo-badge.js` を同一オリジンで配信し、
  他サイト一覧(`config/subutomo-sites.json`)を遅延読み込みします。
  読み込むのはこの静的な一覧のみで、あなたのresult.jsonの内容が外部に
  送られることはありません
- 詳細な設計原則・V1スコープは [docs/spec-viewer.md](docs/spec-viewer.md)、
  今後のロードマップは [docs/plan-v1.md](docs/plan-v1.md) を参照してください
