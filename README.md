# sq-lab

スカッシュのプレイヤー分析データ([squash-lab](https://github.com/t-sumita/squash-lab)
が出力する `result.json`)を閲覧するための、公開の静的ビューアです。コーチが
選手にデータを見せながら指導するための道具として作っています。

**データはこの端末から一切送信されません。** 読み込んだ JSON はブラウザ内で
処理されるだけで、サーバーへのアップロードや外部への通信は行いません。

## 使い方

1. `index.html` をブラウザで開く(GitHub Pages 公開後は URL に直接アクセス)
2. `result.json` をドロップ、またはファイル選択で読み込む
3. 手元にファイルが無い場合は「デモデータを見る」で合成デモデータを確認できる

読み込む `result.json` は [squash-lab](https://github.com/t-sumita/squash-lab)
(動画解析ツール、private リポジトリ)の出力です。sq-lab 自体は動画解析を
行いません。

対応スキーマは `schema_version: "0.2"` のみです。詳細は
[docs/spec-viewer.md](docs/spec-viewer.md) を参照してください。

## 既知の制限

- **T支配率の定義**: T支配率が対象とする T ゾーンは楕円(横1.2m×縦1.0m、
  中心はT交点から0.6m後方)として定義されています。実際の選手の定位置は
  T交点そのものよりやや後方に構えることが多いという競技特性に合わせた
  設定です。このパラメータは読み込んだ `result.json` の
  `metrics_params.t_zone` から動的に表示され、ハードコードされていません。
  また、分母はラリーが行われている時間ではなく、主カメラ区間の合計時間
  です。
- **交差時のID入れ替わり**: squash-lab 側の自動割当は、選手同士が接近・
  交差する瞬間に P1/P2 を取り違えることがあります。誤って割り当てるより
  未割当のまま残す方針のため、一部の断片は「未割当」として除外されます
  (該当する場合は画面上に注記が表示されます)。

## 開発について

- vanilla JS + Canvas、ビルドステップなし(npm 依存なし)
- `demo/demo-match.json` は完全な合成データです(架空のプレイヤー名
  Demo A / Demo B、人工的な位置・指標値)。実際の試合データではありません
- 詳細な設計原則・V1スコープは [docs/spec-viewer.md](docs/spec-viewer.md)
  を参照してください
