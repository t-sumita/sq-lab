# Squash Laboratory (sq-lab)

> ⚠️ **プレリリース (v0.4.0)**: これは知り合い向けのプレリリースです。
> 想定外の不具合が残っている可能性があります。ASISでの提供とし、
> 継続的なサポートは約束できません。

スカッシュのプレイヤー分析データ(`result.json`)を閲覧するための、公開の
静的ビューアです。コーチが選手にデータを見せながら指導するための道具として
作っています。

**データはこの端末から一切送信されません。** 読み込んだ JSON はブラウザ内で
処理されるだけで、サーバーへのアップロードや外部への通信は行いません。

## 使い方

1. `index.html` をブラウザで開く(GitHub Pages 公開後は URL に直接アクセス)
2. `result.json` をドロップ、またはファイル選択で読み込む
3. 手元にファイルが無い場合は「デモデータを見る」で合成デモデータを確認できる

読み込む `result.json` は動画解析ツール(the Analyzer、別リポジトリで開発)の
出力です。本サイト自体は動画解析を行いません。

対応スキーマは `schema_version: "0.2"` のみです。各指標の定義は
[guide.html](guide.html) を、開発者向けの詳細仕様は
[docs/spec-viewer.md](docs/spec-viewer.md) を参照してください。

## 既知の制限

- **交差時のID入れ替わり**: 動画解析ツール側の自動割当は、選手同士が接近・
  交差する瞬間に P1/P2 を取り違えることがあります。誤って割り当てるより
  未割当のまま残す方針のため、一部の断片は「未割当」として除外されます
  (該当する場合は画面上に注記が表示されます)。
- 各指標(Distance / Work Rate / T-Dominance / Coverage / Avg Speed)の
  詳細な定義は [guide.html](guide.html) を参照してください。

## 開発について

- vanilla JS + Canvas、ビルドステップなし(npm 依存なし)
- `demo/demo-match.json` は完全な合成データです(架空のプレイヤー名
  Demo A / Demo B、人工的な位置・指標値)。実際の試合データではありません
- 詳細な設計原則・V1スコープは [docs/spec-viewer.md](docs/spec-viewer.md)、
  今後のロードマップは [docs/plan-v1.md](docs/plan-v1.md) を参照してください
