## v1.1(次期)

1. **印刷対応**: `@media print` + `break-inside` 制御で、コーチが紙で
   選手に渡せる出力に対応する。加えてコンパクト表示モード(縦方向の
   情報密度を上げるトグル)を用意する
2. **複数データ管理**: 閲覧履歴と複数データ切替は**全端末対応**を目標とする。
   - **スマホ**: 読込済みJSONを IndexedDB に保存し、「最近の試合」一覧から
     再オープンできるようにする
   - **PC**: 上記に加えて File System Access API(`showDirectoryPicker`)の
     フォルダ選択を上位互換として提供する。フォルダを開くと、内包する
     result.json 一覧から切替できる。**解析(動画処理)はPC専用**であり、
     この機能はあくまで閲覧済みデータの管理・切替に限る
   - `showDirectoryPicker` は iPhone Safari 非対応のため、単ファイル読込
     (現行)と複数ファイル選択は**恒久的なフォールバック**として維持する。
     混乱を招かないよう、フォルダ機能は「PC向けの上級機能」として
     UI上明確に区別して配置する
   - V2-α の動画同期機能は、スマホでのローカルmp4選択を含めて設計する
     (PCのフォルダ機能に限定しない)
3. **subutomo-badge統合**: サイト共通のバッジ(ロゴ+他サイトへのリンク
   パネル)を組み込む(`C:\dev\subutomo-template` の手順に従う)
4. **guide.html の "How it works" 拡充**: 解析→JSON→ビューアの流れを
   一枚絵で説明する。解析(動画処理)の実行条件も明記する:
   NVIDIA GPU推奨、事前のコート校正が必要、「寝る前に実行して翌朝結果を
   見る」運用が現実的、AS-ISでサポートは提供されない

## V2-α(解析側と連動)

スカッシュ動画解析ツール(the Analyzer、旧称squash-lab)側との連動が
必要な機能。解析側のマイルストーン番号(M6等)は the Analyzer 側の
CC指示書で管理される。

1. **解析側 M6: ラリー終端候補の自動抽出** — 両プレイヤーの移動速度が
   同時に低下・静止する時刻を候補として自動抽出する。あわせて、
   基準フレームのプレイヤー確認用静止画(P1=赤枠/P2=青枠)を出力する。
   これらはスキーマ **0.3** で result.json に追加する(加算的変更)
2. **ビューア側 S5: ラリー区切りマーカーとコーチコメント**
   - タイムラインに▲マーカー(プレイヤー色)を表示する
   - ▲マーカーは**勝者(プレイヤー色)の意味を持つ**。時系列に集計する
     ことでスコアを自動算出し、リプレイ・動画にスコアを重畳表示する
     (例: `7-5`)。ゲームの区切りは11点到達時点で自動提案し、手動調整も
     可能にする。副産物として、ラリー長の分布やサーブ側の勝率が算出
     できるようになる
   - マーカーは人手で追加/削除/時刻調整できる。時刻に紐づくコーチ
     コメントも入力できる
   - これらは **annotations.json** として書き出し/読み込みできる
     (result.json と対で配布可能な、独立したファイルとする)
3. **ローカルmp4読込と`<video>`のタイムライン同期** — 通信ゼロ原則は
   維持したまま、選手が手元のmp4ファイルを選択し、リプレイのタイムライン
   と動画再生を同期させる

## backlog(着手時期未定)

- **リポジトリ名の見直し**: `squash-lab` → 利用者に伝わる名称
  (例: the Analyzer に対応する名前)へ。GitHubのリネームは旧URLからの
  リダイレクトが効くため、Phase 2 着手時に改めて判断する
- **T-zoneのproプリセット**: `a_m=1.5 / b_m=1.2 / offset_m=0.9`
  (M5のtzone_sweep結果に残っている候補)をプリセットとして選べるようにする
- **ガウス減衰スコアリング**: T支配率等の指標を、楕円の内外で二値判定する
  現行方式から、中心からの距離に応じた連続的な重み付け(ガウス減衰)へ
  発展させる案の検討
- **他競技プロファイル構想**: パデル・バドミントン等、スカッシュ以外の
  競技への展開。コート寸法・ゾーン定義・指標をハードコードではなく
  **設定駆動**にする。**今後、競技固有の値を新たにハードコードしない
  ことを実装原則とする**(court.js等の既存ハードコードを直ちに置き換える
  ことまでは求めないが、新規追加分から適用する)
- **スカッシュ界への貢献ビジョン**: スカッシュ業界は狭く、プロ選手本人が
  自身の試合動画を提供してくれる可能性がある。配布はツールが関与せず、
  私的チャネル(LINE等)の信頼関係に委ねる現方針を維持する
  (自動配布・公開ギャラリー等の機能は作らない)
